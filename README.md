# Portfolio Web

The public Portfolio website — manifesto, criteria, principles, doors.

Intended canonical domain: **portfolio.sc**.

## Contents

- `index.html` — single-page site (light mode default, dark toggle)
- `favicon.svg` — 32×32 P-mark favicon
- `README.md` — this file

## Design

Built directly on the Portfolio brand DNA (see `Studio/Brand/`):

- Momo Trust Display for headlines, wordmark, pull quotes
- Inter for body and UI
- Light mode (warm off-white `#F6F4EE`) default; dark mode (`#0A0A0A`) toggle
- Negative letter-spacing as signature (-5% display, -4% leads, -2.5% body)
- 0.5px hairlines between sections
- 12px border-radius on the doors grid

No images, no project list, no team page. The site is a manifesto with a door.

## Structure

1. Hero — tagline + the two-forces stake
2. The bet — why now
3. The model — small fast teams, live iteration, spin-outs
4. What we look for — the six criteria
5. How we operate — the five principles
6. Where ideas come from — no pride of ownership
7. Get involved — four doors (lead, contribute, bring an idea, partner)

## Deploy

Drop this folder onto a Vercel project pointed at `portfolio.sc`. No build step required.

The footer links to `studio.portfolio.sc` for the brand guide.
